## Обзор

Otedama - это полностью автоматизированная коммерческая платформа P2P майнинг-пула, DEX и DeFi. Построена в соответствии с философиями проектирования Джона Кармака (производительность прежде всего), Роберта К. Мартина (чистая архитектура) и Роба Пайка (простота).

### Ключевые особенности

- **Полностью автоматическая работа** - Не требует ручного вмешательства
- **Неизменяемая система комиссий** - Неизменяемый автоматический сбор 1,5% в BTC
- **Поддержка мульти-алгоритмов** - Совместимость с CPU/GPU/ASIC
- **Унифицированный DEX** - V2 AMM + V3 концентрированная ликвидность
- **Автоматические выплаты** - Автоматические почасовые выплаты вознаграждений майнерам
- **Функции DeFi** - Автоликвидация, управление, мосты
- **Корпоративный уровень** - Поддержка 10,000+ майнеров

### Функции операционной автоматизации

1. **Автоматический сбор комиссий**
   - BTC адрес: 1GzHriuokSrZYAZEEWoL7eeCCXsX3WyLHa (неизменяемый)
   - Комиссия пула: 0% (удалена)
   - Операционная комиссия: 1,5% (неизменяемая)
   - Общая комиссия: 1,5% (только операционная комиссия)
   - Частота сбора: Каждые 5 минут
   - Автоматическая конвертация всех валют в BTC

2. **Автоматическое распределение вознаграждений за майнинг**
   - Выполняется каждый час
   - Автоматическое вычитание комиссий пула
   - Автоматическая отправка при достижении минимальной выплаты
   - Автоматическая запись транзакций

3. **Полностью автоматизированные DEX/DeFi**
   - Автоматическая ребалансировка пулов ликвидности
   - Автоликвидация (85% LTV)
   - Автоматическое выполнение предложений управления
   - Автоматическая ретрансляция кроссчейн мостов

## Установка

### 1. Базовая установка

```bash
# Клонировать репозиторий
git clone https://github.com/otedama/otedama.git
cd otedama

# Установить зависимости
npm install

# Запуск
npm start
```

### 2. Установка Docker

```bash
# Запуск с Docker Compose
docker-compose up -d

# Проверка логов
docker-compose logs -f otedama
```

### 3. Установка в один клик

**Windows:**
```batch
.\quickstart.bat
```

**Linux/macOS:**
```bash
chmod +x quickstart.sh
./quickstart.sh
```

## Конфигурация

### Базовая конфигурация

Редактировать `otedama.json`:

```json
{
  "pool": {
    "name": "Название вашего пула",
    "fee": 1.0,
    "minPayout": {
      "BTC": 0.001,
      "RVN": 100,
      "XMR": 0.1
    }
  },
  "mining": {
    "currency": "RVN",
    "algorithm": "kawpow",
    "walletAddress": "Адрес вашего кошелька"
  }
}
```

### Конфигурация командной строки

```bash
# Базовый запуск
node index.js --wallet RYourWalletAddress --currency RVN

# Высокая производительность
node index.js --threads 16 --max-miners 5000 --enable-dex

# Пользовательские порты
node index.js --api-port 9080 --stratum-port 4444
```

## Подключение майнеров

### Информация о подключении
- Сервер: `ВАШ_IP:3333`
- Имя пользователя: `АдресКошелька.ИмяВоркера`
- Пароль: `x`

### Примеры программ для майнинга

**T-Rex (NVIDIA):**
```bash
t-rex -a kawpow -o stratum+tcp://ВАШ_IP:3333 -u RWallet.worker1 -p x
```

**TeamRedMiner (AMD):**
```bash
teamredminer -a kawpow -o stratum+tcp://ВАШ_IP:3333 -u RWallet.worker1 -p x
```

**XMRig (CPU):**
```bash
xmrig -o ВАШ_IP:3333 -u 4MoneroWallet -p x -a rx/0
```

## Информация для операторов

### Структура доходов

1. **Операционная комиссия**: 1,5% фиксированная (неизменяемая)
2. **Комиссия DEX**: 0,3% (распределяется поставщикам ликвидности)
3. **Комиссия DeFi**: Часть кредитных процентов

### Автоматизированные задачи

- **Каждые 5 минут**: Конвертация и сбор операционной комиссии в BTC
- **Каждые 10 минут**: Ребалансировка пулов DEX
- **Каждые 30 минут**: Проверка ликвидации DeFi
- **Каждый час**: Автоматические выплаты майнерам
- **Каждые 24 часа**: Оптимизация базы данных и резервное копирование

### Мониторинг

Панель управления: `http://localhost:8080`

Ключевые метрики:
- Активные майнеры
- Хешрейт
- Доход от комиссий
- Объем DEX
- Системные ресурсы
