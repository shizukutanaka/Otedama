## ภาพรวม

Otedama เป็นแพลตฟอร์ม P2P mining pool, DEX และ DeFi ระดับเชิงพาณิชย์ที่ทำงานอัตโนมัติเต็มรูปแบบ สร้างขึ้นตามปรัชญาการออกแบบของ John Carmack (ประสิทธิภาพเป็นอันดับแรก), Robert C. Martin (สถาปัตยกรรมที่สะอาด) และ Rob Pike (ความเรียบง่าย)

### คุณสมบัติหลัก

- **การทำงานอัตโนมัติเต็มรูปแบบ** - ไม่ต้องมีการแทรกแซงด้วยตนเอง
- **ระบบค่าธรรมเนียมที่ไม่เปลี่ยนแปลง** - การเรียกเก็บ BTC อัตโนมัติ 1.5% ที่ไม่สามารถแก้ไขได้
- **รองรับหลายอัลกอริทึม** - เข้ากันได้กับ CPU/GPU/ASIC
- **DEX แบบรวม** - V2 AMM + V3 Concentrated Liquidity
- **การชำระเงินอัตโนมัติ** - จ่ายรางวัลให้นักขุดอัตโนมัติทุกชั่วโมง
- **คุณสมบัติ DeFi** - การชำระบัญชีอัตโนมัติ, การกำกับดูแล, สะพานเชื่อมต่อ
- **ระดับองค์กร** - รองรับนักขุด 10,000+ คน

### คุณสมบัติการทำงานอัตโนมัติ

1. **การเรียกเก็บค่าธรรมเนียมอัตโนมัติ**
   - ที่อยู่ BTC: 1GzHriuokSrZYAZEEWoL7eeCCXsX3WyLHa (ไม่เปลี่ยนแปลง)
   - ค่าธรรมเนียม Pool: 0% (ยกเลิกแล้ว)
   - ค่าธรรมเนียมการดำเนินงาน: 1.5% (ไม่สามารถแก้ไขได้)
   - ค่าธรรมเนียมรวม: 1.5% (เฉพาะค่าธรรมเนียมการดำเนินงาน)
   - ความถี่ในการเรียกเก็บ: ทุก 5 นาที
   - แปลงสกุลเงินทั้งหมดเป็น BTC อัตโนมัติ

2. **การแจกจ่ายรางวัลการขุดอัตโนมัติ**
   - ดำเนินการทุกชั่วโมง
   - หักค่าธรรมเนียม pool อัตโนมัติ
   - ส่งอัตโนมัติเมื่อถึงจำนวนขั้นต่ำ
   - บันทึกธุรกรรมอัตโนมัติ

3. **DEX/DeFi อัตโนมัติเต็มรูปแบบ**
   - ปรับสมดุล liquidity pools อัตโนมัติ
   - การชำระบัญชีอัตโนมัติ (85% LTV)
   - ดำเนินการข้อเสนอการกำกับดูแลอัตโนมัติ
   - ส่งต่อ cross-chain bridges อัตโนมัติ

## การติดตั้ง

### 1. การติดตั้งพื้นฐาน

```bash
# โคลน repository
git clone https://github.com/otedama/otedama.git
cd otedama

# ติดตั้ง dependencies
npm install

# เริ่มต้น
npm start
```

### 2. การติดตั้ง Docker

```bash
# เริ่มต้นด้วย Docker Compose
docker-compose up -d

# ตรวจสอบ logs
docker-compose logs -f otedama
```

### 3. การติดตั้งแบบคลิกเดียว

**Windows:**
```batch
.\quickstart.bat
```

**Linux/macOS:**
```bash
chmod +x quickstart.sh
./quickstart.sh
```

## การตั้งค่า

### การตั้งค่าพื้นฐาน

แก้ไข `otedama.json`:

```json
{
  "pool": {
    "name": "ชื่อ Pool ของคุณ",
    "fee": 1.0,
    "minPayout": {
      "BTC": 0.001,
      "RVN": 100,
      "XMR": 0.1
    }
  },
  "mining": {
    "currency": "RVN",
    "algorithm": "kawpow",
    "walletAddress": "ที่อยู่กระเป๋าเงินของคุณ"
  }
}
```

### การตั้งค่า Command Line

```bash
# เริ่มต้นพื้นฐาน
node index.js --wallet RYourWalletAddress --currency RVN

# ประสิทธิภาพสูง
node index.js --threads 16 --max-miners 5000 --enable-dex

# พอร์ตแบบกำหนดเอง
node index.js --api-port 9080 --stratum-port 4444
```

## การเชื่อมต่อนักขุด

### ข้อมูลการเชื่อมต่อ
- เซิร์ฟเวอร์: `IP_ของคุณ:3333`
- ชื่อผู้ใช้: `ที่อยู่กระเป๋าเงิน.ชื่อ Worker`
- รหัสผ่าน: `x`

### ตัวอย่างซอฟต์แวร์การขุด

**T-Rex (NVIDIA):**
```bash
t-rex -a kawpow -o stratum+tcp://IP_ของคุณ:3333 -u RWallet.worker1 -p x
```

**TeamRedMiner (AMD):**
```bash
teamredminer -a kawpow -o stratum+tcp://IP_ของคุณ:3333 -u RWallet.worker1 -p x
```

**XMRig (CPU):**
```bash
xmrig -o IP_ของคุณ:3333 -u 4MoneroWallet -p x -a rx/0
```

## ข้อมูลสำหรับผู้ดำเนินการ

### โครงสร้างรายได้

1. **ค่าธรรมเนียมการดำเนินงาน**: 1.5% คงที่ (ไม่สามารถแก้ไขได้)
2. **ค่าธรรมเนียม DEX**: 0.3% (แจกจ่ายให้ผู้ให้สภาพคล่อง)
3. **ค่าธรรมเนียม DeFi**: ส่วนหนึ่งของดอกเบี้ยเงินกู้

### งานอัตโนมัติ

- **ทุก 5 นาที**: แปลงและเรียกเก็บค่าธรรมเนียมการดำเนินงานเป็น BTC
- **ทุก 10 นาที**: ปรับสมดุล DEX pool
- **ทุก 30 นาที**: ตรวจสอบการชำระบัญชี DeFi
- **ทุกชั่วโมง**: จ่ายเงินอัตโนมัติให้นักขุด
- **ทุก 24 ชั่วโมง**: เพิ่มประสิทธิภาพฐานข้อมูลและสำรองข้อมูล

### การตรวจสอบ

แดชบอร์ด: `http://localhost:8080`

ตัวชี้วัดสำคัญ:
- นักขุดที่ใช้งานอยู่
- อัตรา Hash
- รายได้ค่าธรรมเนียม
- ปริมาณ DEX
- ทรัพยากรระบบ

## ความปลอดภัย

### การป้องกันที่นำมาใช้

1. **การป้องกัน DDoS**
   - การจำกัดอัตราหลายระดับ
   - เกณฑ์ที่ปรับตัวได้
   - Challenge-response

2. **ระบบการรับรองความถูกต้อง**
   - JWT + MFA
   - การควบคุมการเข้าถึงตามบทบาท
   - การจัดการคีย์ API

3. **การป้องกันการปลอมแปลง**
   - ที่อยู่ค่าธรรมเนียมการดำเนินงานที่ไม่เปลี่ยนแปลง
   - การตรวจสอบความสมบูรณ์ของระบบ
   - บันทึกการตรวจสอบ
