## अवलोकन

Otedama एक पूर्ण स्वचालित वाणिज्यिक-ग्रेड P2P माइनिंग पूल, DEX और DeFi प्लेटफॉर्म है। John Carmack (प्रदर्शन पहले), Robert C. Martin (स्वच्छ आर्किटेक्चर) और Rob Pike (सरलता) की डिज़ाइन दर्शन के अनुसार बनाया गया।

### मुख्य विशेषताएं

- **पूर्ण स्वचालित संचालन** - कोई मैन्युअल हस्तक्षेप आवश्यक नहीं
- **अपरिवर्तनीय शुल्क प्रणाली** - गैर-संशोधनीय 1.5% BTC स्वचालित संग्रह
- **मल्टी-एल्गोरिथम समर्थन** - CPU/GPU/ASIC संगत
- **एकीकृत DEX** - V2 AMM + V3 केंद्रित तरलता
- **स्वचालित भुगतान** - प्रति घंटे स्वचालित माइनर पुरस्कार भुगतान
- **DeFi सुविधाएं** - स्वतः-परिसमापन, शासन, ब्रिज
- **एंटरप्राइज़ ग्रेड** - 10,000+ माइनर्स का समर्थन

### परिचालन स्वचालन सुविधाएं

1. **स्वचालित शुल्क संग्रह**
   - BTC पता: 1GzHriuokSrZYAZEEWoL7eeCCXsX3WyLHa (अपरिवर्तनीय)
   - पूल शुल्क: 0% (हटाया गया)
   - परिचालन शुल्क: 1.5% (गैर-संशोधनीय)
   - कुल शुल्क: 1.5% (केवल परिचालन शुल्क)
   - संग्रह आवृत्ति: हर 5 मिनट
   - सभी मुद्राओं को BTC में स्वचालित रूपांतरण

2. **स्वचालित माइनिंग पुरस्कार वितरण**
   - प्रति घंटे निष्पादन
   - स्वचालित पूल शुल्क कटौती
   - न्यूनतम भुगतान पर पहुंचने पर स्वतः भेजना
   - स्वचालित लेनदेन रिकॉर्डिंग

3. **पूर्ण स्वचालित DEX/DeFi**
   - तरलता पूल का स्वचालित पुनर्संतुलन
   - स्वतः-परिसमापन (85% LTV)
   - शासन प्रस्तावों का स्वचालित निष्पादन
   - क्रॉस-चेन ब्रिज का स्वचालित रिले

## स्थापना

### 1. बुनियादी स्थापना

```bash
# रिपॉजिटरी क्लोन करें
git clone https://github.com/otedama/otedama.git
cd otedama

# निर्भरताएं स्थापित करें
npm install

# शुरू करें
npm start
```

### 2. Docker स्थापना

```bash
# Docker Compose के साथ शुरू करें
docker-compose up -d

# लॉग जांचें
docker-compose logs -f otedama
```

### 3. एक-क्लिक स्थापना

**Windows:**
```batch
.\quickstart.bat
```

**Linux/macOS:**
```bash
chmod +x quickstart.sh
./quickstart.sh
```

## कॉन्फ़िगरेशन

### बुनियादी कॉन्फ़िगरेशन

`otedama.json` संपादित करें:

```json
{
  "pool": {
    "name": "आपके पूल का नाम",
    "fee": 1.0,
    "minPayout": {
      "BTC": 0.001,
      "RVN": 100,
      "XMR": 0.1
    }
  },
  "mining": {
    "currency": "RVN",
    "algorithm": "kawpow",
    "walletAddress": "आपका वॉलेट पता"
  }
}
```

### कमांड लाइन कॉन्फ़िगरेशन

```bash
# बुनियादी स्टार्टअप
node index.js --wallet RYourWalletAddress --currency RVN

# उच्च प्रदर्शन
node index.js --threads 16 --max-miners 5000 --enable-dex

# कस्टम पोर्ट
node index.js --api-port 9080 --stratum-port 4444
```

## माइनर कनेक्शन

### कनेक्शन जानकारी
- सर्वर: `आपका_IP:3333`
- उपयोगकर्ता नाम: `वॉलेटएड्रेस.वर्करनाम`
- पासवर्ड: `x`

### माइनिंग सॉफ्टवेयर उदाहरण

**T-Rex (NVIDIA):**
```bash
t-rex -a kawpow -o stratum+tcp://आपका_IP:3333 -u RWallet.worker1 -p x
```

**TeamRedMiner (AMD):**
```bash
teamredminer -a kawpow -o stratum+tcp://आपका_IP:3333 -u RWallet.worker1 -p x
```

**XMRig (CPU):**
```bash
xmrig -o आपका_IP:3333 -u 4MoneroWallet -p x -a rx/0
```

## ऑपरेटर जानकारी

### राजस्व संरचना

1. **परिचालन शुल्क**: 1.5% निश्चित (गैर-संशोधनीय)
2. **DEX शुल्क**: 0.3% (तरलता प्रदाताओं को वितरित)
3. **DeFi शुल्क**: ऋण ब्याज का हिस्सा

### स्वचालित कार्य

- **हर 5 मिनट**: परिचालन शुल्क BTC रूपांतरण और संग्रह
- **हर 10 मिनट**: DEX पूल पुनर्संतुलन
- **हर 30 मिनट**: DeFi परिसमापन जांच
- **हर घंटे**: स्वचालित माइनर भुगतान
- **हर 24 घंटे**: डेटाबेस अनुकूलन और बैकअप

डैशबोर्ड: `http://localhost:8080`

- सक्रिय माइनर
- हैशरेट
- शुल्क राजस्व
- DEX वॉल्यूम

## सुरक्षा

### लागू सुरक्षा

1. **DDoS सुरक्षा**
   - बहु-स्तरीय दर सीमा
   - अनुकूली थ्रेसहोल्ड
   - चुनौती-प्रतिक्रिया

2. **प्रमाणीकरण प्रणाली**
   - JWT + MFA
   - भूमिका-आधारित पहुंच नियंत्रण
   - API कुंजी प्रबंधन

3. **छेड़छाड़ रोकथाम**
   - अपरिवर्तनीय परिचालन शुल्क पता
   - सिस्टम अखंडता जांच
   - ऑडिट लॉग
