## Преглед

Otedama е напълно автоматизирана търговска P2P майнинг пул, DEX и DeFi платформа. Изградена следвайки дизайнерските философии на John Carmack (производителност на първо място), Robert C. Martin (чиста архитектура) и Rob Pike (простота).

### Ключови Характеристики

- **Напълно Автоматизирана Работа** - Нулева ръчна намеса
- **Непроменима Система за Такси** - Немодифицируемо автоматично събиране на 0,1% BTC
- **Поддръжка на Мулти-Алгоритми** - Съвместим с CPU/GPU/ASIC
- **Единен DEX** - V2 AMM + V3 Концентрирана Ликвидност
- **Автоматично Плащане** - Почасови автоматични награди за майнери
- **DeFi Функции** - Авто-ликвидация, управление, мостове
- **Корпоративно Ниво** - Поддържа 10,000+ майнери

### Функции за Оперативна Автоматизация

1. **Автоматично Събиране на Такси**
   - BTC Адрес: Хардкодиран (непроменим)
   - Такса на Пула: 1,4% (немодифицируема)
   - Оперативна Такса: 0,1% (немодифицируема)
   - Обща Такса: 1,5% (напълно фиксирана)
   - Честота на Събиране: На всеки 5 минути
   - Автоматично конвертира всички валути в BTC

2. **Автоматично Разпределение на Майнинг Награди**
   - Изпълнява се всеки час
   - Автоматично приспада таксите на пула
   - Автоматично изпраща при достигане на минимално плащане
   - Автоматично записва транзакции

3. **Напълно Автоматизиран DEX/DeFi**
   - Авто-балансиране на пулове за ликвидност
   - Авто-ликвидация (85% LTV)
   - Авто-изпълнение на предложения за управление
   - Авто-препредаване на cross-chain мостове

## Инсталация

### 1. Основна Инсталация

```bash
# Клониране на хранилището
git clone https://github.com/otedama/otedama.git
cd otedama

# Инсталиране на зависимости
npm install

# Стартиране
npm start
```

### 2. Docker Инсталация

```bash
# Стартиране с Docker Compose
docker-compose up -d

# Проверка на логове
docker-compose logs -f otedama
```

### 3. Инсталация с Едно Кликване

**Windows:**
```batch
.\quickstart.bat
```

**Linux/macOS:**
```bash
chmod +x quickstart.sh
./quickstart.sh
```

## Конфигурация

### Основна Конфигурация

Редактирайте `otedama.json`:

```json
{
  "pool": {
    "name": "Името на Вашия Пул",
    "fee": 1.0,
    "minPayout": {
      "BTC": 0.001,
      "RVN": 100,
      "XMR": 0.1
    }
  },
  "mining": {
    "currency": "RVN",
    "algorithm": "kawpow",
    "walletAddress": "Вашият Адрес на Портфейла"
  }
}
```

### Конфигурация от Командния Ред

```bash
# Основно стартиране
node index.js --wallet RYourWalletAddress --currency RVN

# Висока производителност
node index.js --threads 16 --max-miners 5000 --enable-dex

# Персонализирани портове
node index.js --api-port 9080 --stratum-port 4444
```

## Връзка на Майнери

### Информация за Връзка
- Сървър: `ВАШЕТО_IP:3333`
- Потребителско име: `АдресПортфейл.ИмеРаботник`
- Парола: `x`

### Примери за Майнинг Софтуер

**T-Rex (NVIDIA):**
```bash
t-rex -a kawpow -o stratum+tcp://ВАШЕТО_IP:3333 -u RWallet.worker1 -p x
```

**TeamRedMiner (AMD):**
```bash
teamredminer -a kawpow -o stratum+tcp://ВАШЕТО_IP:3333 -u RWallet.worker1 -p x
```

**XMRig (CPU):**
```bash
xmrig -o ВАШЕТО_IP:3333 -u 4MoneroWallet -p x -a rx/0
```

## Информация за Оператори

### Структура на Приходите

1. **Такса на Пула**: 1,4% фиксирана (немодифицируема)
2. **Оперативна Такса**: 0,1% фиксирана (немодифицируема)
3. **Обща Майнинг Такса**: 1,5% (напълно фиксирана)
4. **DEX Такса**: 0,3% (разпределена към доставчиците на ликвидност)
5. **DeFi Такса**: Част от лихвите по заеми

### Автоматизирани Задачи

- **На всеки 5 минути**: Конвертиране и събиране на оперативна такса BTC
- **На всеки 10 минути**: Балансиране на DEX пул
- **На всеки 30 минути**: Проверка на DeFi ликвидация
- **Всеки час**: Автоматични плащания на майнери
- **На всеки 24 часа**: Оптимизация и архивиране на база данни

### Мониторинг

Табло за управление: `http://localhost:8080`

Ключови Показатели:
- Активни майнери
- Hash скорост
- Приходи от такси
- DEX обем
- Системни ресурси
